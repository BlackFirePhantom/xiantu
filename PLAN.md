## 一、项目概述

**仙途**是一款轻量级多人在线修仙文字 RPG，基于 Flask + Flask-SocketIO + SQLite 构建。玩家从青云镇出发，历经九大险地，通过斩妖、修炼、炼丹、锻造等玩法，最终渡劫飞升。

**定位**：适合 2 核 1G 低配服务器部署的休闲放置修仙游戏，支持 50+ 人同时在线。

## 六、技术决策建议

### 6.1 保持还是换框架？

**建议：保持当前技术栈不变。**

理由：
- 项目定位是轻量级游戏，Flask + SQLite 完全够用
- 迁移到 Django/FastAPI 的收益不大，成本很高
- 前端已模块化拆分（socket.js/ui.js/main.js），无需引入 Vue/React
- SQLite 在 50 人并发下表现优异，无需迁移到 PostgreSQL

### 6.2 性能瓶颈预判

| 场景 | 当前承载 | 瓶颈点 | 解决方案 |
|------|----------|--------|----------|
| 同时在线 | 50+ | SocketIO 线程模式 | 若超过 100 人，考虑切换 eventlet/gevent |
| 数据库 | 无限 | SQLite 写锁 | WAL 模式已启用；若写冲突频繁，加写队列 |
| 内存 | ~50MB | 字典缓存（online_users） | 定期清理离线用户 |
| 文件大小 | 单文件 DB | game.db 超 100MB | 定期 VACUUM，归档旧数据 |

### 6.3 安全加固优先级

1. ~~**立即修复**：密码加盐哈希~~ ✅ 已完成
2. ~~**尽快修复**：固定 secret_key、CORS 限制~~ ✅ 已完成
3. **计划修复**：CSRF 防护、SocketIO 认证、输入校验
4. **可选增强**：登录频率限制、IP 封禁、管理员操作审计
